RagaM-Roll — Help

RagaM-Roll — a play on "Rock and Roll" — is a Carnatic-notation playground. Type srgm notation on the top left; see the textbook-style line and the ragamroll (a pianoroll-style roll — the app's signature view) on the top right; hear it, retune it, and browse ragas & talas.

The panes

  • Editor (left): your srgm source. It auto-compiles as you type (off the main thread, so it never freezes).
  • Ragamroll (top-right): the ascii roll; the playing row highlights during playback.
  • Notation (bottom): the Carnatic-textbook single-line form with barlines.
  • Drag the dividers between panes to resize.

Writing srgm — the melody

The melody uses a deliberately tiny alphabet so it stays terse:

TokenMeaning
s r g m p d nThe seven swaras. Case picks the variety — lower vs upper is the komal/shuddha (lower/higher) form, resolved against the current raga.
> / <Prepend to a swara to raise / lower the octave, e.g. >s, <p. The shift persists for the notes that follow.
2, 4, …An integer after a swara sets its length, e.g. s4. Default is 1.
zA rest (silence). Takes a length too: z2.
-A separator (visual grouping; no time).
%A line starting with % is a comment.

Cutting repetition

DirectiveMeaning
L=2Default note-length multiplier: absLen = L × the number on the swara. So S2 G2 M2 P2 N4 can be written L=2 S G M P N2.
O=5Set the absolute octave register.

Directives — the setup line

These appear once or a few times, so they read verbosely for clarity:

DirectiveMeaning
Raga=mohanam,0The raga (name) and a semitone transpose of Sa. The raga fixes how each swara letter resolves to a pitch.
Tala=adi,4The tala (cycle) and a length resolution — chosen so the shortest note in the piece is an integer 1 (one akshara = N length-units).
T120Tempo in BPM.
I[FLUTE]Instrument (General-MIDI name) for export.

Example

% A short Mohanam phrase
Raga=mohanam,0  Tala=rupaka,2  T160
L=1 O=5
G2 G2 P4 P4  D2 P2 >S4 S4

Playing & the mixer

The transport row (above the panes) holds:

  • ▶ ⏸ ⏹ play / pause / stop. The ragamroll highlights the current row as it plays.
  • Sa the reference pitch — Auto follows the raga, or pin Sa to a note (C4–C6). Transposes playback.
  • ♩ tempo a speed slider (0.2× steps) plus an exact-BPM box; ↺ returns to the composition tempo.
  • 🎶 Drone, 🎼 Melody, 🔊 Tala, 🔈 Master — each icon is a mute toggle (keeps its set level); sliders set volume. Mute melody to solo tala + drone.
  • 🎻 Timbre the melody voice: Soft, Bowed, or Reed.

Volumes are live during playback. Tempo, Sa transpose, and the scale override apply on the next Play.

Open, Save, Export

  • Open ▾ — load a bundled example or a .srgm file from your device.
  • Save — download your source as .srgm.
  • Export MIDI — a standard .mid of the melody (12-EDO; the microtonal scale override is web-audio only).

Ragas & Talas

  • Ragas — a searchable reference of the raga scales (swaras, intervals in semitones).
  • Talas — each tala as a row of akshara boxes with accents marked (X = sam, o = anga start). ▶ plays the cycle as strums and lights each box in time.

Scale ⚙ — experimental microtonal tuning

Override the playback pitches with an authentic 53-EDO tuning built from the 72 melakartas:

  • Pick a mela (grouped by the 12 chakras) — or tap Match piece to jump to the mela matching your raga.
  • Each of R, G, M, D, N shows its variety and an a / b comma toggle — the two microtonal shrutis a comma apart. Defaults to the just-intonation-nearest.
  • The table shows the resulting swaras, 53-EDO steps, and cents. Apply retunes playback; ↺ returns to the plain 12-EDO ragabase.

The tuning affects web-audio playback and the drone only — notation and MIDI export stay unchanged.

Offline

RagaM-Roll is a PWA: after the first visit it works fully offline, and your last composition is remembered in the browser. Add it to your home screen to launch it like an app.
